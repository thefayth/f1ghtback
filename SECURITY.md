# Security

## Reporting

Do not open a public issue containing personal records, court records, credentials, private screenshots, or vulnerability details that could expose a user. Use GitHub's private vulnerability reporting when available.

## Data Boundary

The public application is intentionally limited to enumerated, non-personal selections. It does not need names, case numbers, narratives, documents, or accounts. Do not extend the public input surface to collect those fields without a separate threat model and privacy review.

Never commit `.env` files, API keys, tokens, Sites credentials, private records, screenshots, transcripts, `.fayth` objects, or local storage.
